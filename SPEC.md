# 农产品销售管理微信小程序 - 项目规格文档 (SPEC.md)

## 1. 产品概述

### 1.1 项目背景与目标
开发一个微信小程序，用于家庭农产品（车厘子、枇杷等）的订单管理、发货跟踪及营销运营。核心解决发货人统计汇总、照片存档、物流跟踪问题，支持多角色协作。

### 1.2 目标用户
- **果园所有者**：全局管理者，配置商品、价格、优惠券，查看年度统计
- **填写人**：家庭成员（3人可同时操作），填写客户订单
- **发货人**：接收订单并执行发货操作
- **外部浏览者**：无需登录查看商品和物流跟踪

## 2. 核心功能模块

### 2.1 模块A：商品与价格配置（仅果园所有者）
| 功能编号 | 功能名称 | 详细描述 |
|---------|---------|---------|
| A1 | 产品种类管理 | 增删改农产品（车厘子、枇杷等），设置熟制周期 |
| A2 | 规格设置 | 每种产品下设置规格（大/中/小果，2斤/5斤装），绑定单价 |
| A3 | 邮费规则引擎 | 按地区/规格设置邮费，支持包邮规则 |

### 2.2 模块B：订单填写与流转
| 功能编号 | 功能名称 | 详细描述 |
|---------|---------|---------|
| B1 | 订单表单 | 收货地址、联系人、电话、产品、规格、数量、备注 |
| B2 | 实时计价 | 自动计算单价×数量 + 智能邮费 + 优惠券抵扣 |
| B3 | 自动推送 | 提交后推送服务通知到发货人微信 |
| B4 | 多填写人支持 | 3人可同时操作，订单标记填写人身份 |

### 2.3 模块C：发货人工作台
| 功能编号 | 功能名称 | 详细描述 |
|---------|---------|---------|
| C1 | 待办看板 | 接收所有待发货订单，支持状态筛选 |
| C2 | 智能汇总 | 按"产品+规格"自动聚合统计 |
| C3 | 双照片上传 | 水果照片 + 订单页面截图（必填） |
| C4 | 物流曲线 | 上传后显示物流历史时间轴 |
| C5 | 发货确认 | 上传照片后标记发货，填写物流信息 |

### 2.4 模块D：物流跟踪与通知
| 功能编号 | 功能名称 | 详细描述 |
|---------|---------|---------|
| D1 | 订阅机制 | 发货、状态变更、异常提醒推送 |
| D2 | 查询功能 | 订单号/手机号后四位查询物流轨迹 |
| D3 | 授权查看 | 生成链接/二维码分享特定订单物流 |

### 2.5 模块E：优惠券与营销系统
| 功能编号 | 功能名称 | 详细描述 |
|---------|---------|---------|
| E1 | 优惠券类型 | 新用户券、晒单好评券、邀请返利券 |
| E2 | 邀请返利规则 | 阶梯奖励（1→5元, 2→8元, 3→10元, 4+→12元） |
| E3 | 售后工单 | 上传3张图提交问题反馈 |

### 2.6 模块F：数据统计
| 功能编号 | 功能名称 | 详细描述 |
|---------|---------|---------|
| F1 | 年度经营看板 | 各品类销量、规格利润、年度订单总价 |
| F2 | 优惠券统计 | 核销率、邀请返利ROI |
| F3 | 发货效率 | 平均发货时效、待发货积压预警 |

## 3. 技术架构

### 3.1 技术栈
- **平台**：微信小程序（Uni-app跨平台框架）
- **后端**：微信云开发（云数据库 + 云函数 + 云存储）
- **物流API**：快递100
- **图表**：ECharts
- **图片处理**：Canvas水印 + wx.chooseMedia

### 3.2 数据库集合设计
```
users：用户角色、openid、农场绑定
products：产品信息、规格、单价、熟制周期
shipping_rules：地区、规格、邮费规则
orders：订单主表（状态、金额、地址、照片URL、物流信息）
coupons：优惠券（类型、金额、持有人、使用状态）
invites：邀请关系（邀请人、被邀请人、奖励状态）
```

### 3.3 路由结构
```
/pages/index/index - 发货人工作台（首页）
/pages/products/list - 商品列表
/pages/products/detail - 商品详情
/pages/products/config - 商品配置（果园所有者）
/pages/orders/create - 订单填写
/pages/orders/detail - 订单详情
/pages/orders/logistics - 物流跟踪
/pages/orders/track - 物流查询
/pages/coupons/list - 优惠券列表
/pages/stats/dashboard - 数据统计看板
/pages/mine/index - 个人中心
```

## 4. UI/UX设计规范

### 4.1 设计风格
- **主色调**：#E74C3C（樱桃红）+ #F39C12（枇杷黄）
- **辅助色**：#27AE60（自然绿）、#ECF0F1（浅灰白）
- **字体**：系统字体，标题16-18px，正文14px
- **布局**：卡片式布局，简洁清晰，3步完成订单流程
- **图标**：线性图标风格

### 4.2 核心页面设计

#### 发货人工作台（首页）
- 顶部：待办数量大字显示 + 规格汇总卡片
- 中部：待发货订单列表（状态筛选Tab）
- 底部：快捷操作按钮

#### 订单填写页
- Step 1：选择产品和规格
- Step 2：填写收货地址
- Step 3：确认订单并支付

#### 照片上传组件
- 显示示例图指引
- 提示拍摄要求（水果照片、订单截图）
- 支持相机拍照和相册选择

#### 物流曲线
- 纵向时间轴设计
- 当前状态节点高亮
- 快递轨迹可视化展示

## 5. 开发优先级

### P0 阶段（MVP）
- ✅ 模块A基础配置
- ✅ 模块B订单填写
- ✅ 模块C发货工作台（不含曲线）
- ✅ 基础物流查询

### P1 阶段
- ✅ 模块D订阅通知
- ✅ 模块C历史曲线
- ✅ 授权查看功能

### P2 阶段
- ✅ 模块E优惠券 + 邀请返利
- ✅ 模块F统计看板

## 6. 合规要求

- 邀请返利仅限一级（A邀请B，B下单A得券）
- 禁止多级分销
- 用户隐私保护（地址信息加密存储）
- 微信服务通知合规使用
